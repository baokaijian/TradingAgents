产品化设计说明

A 股与港股自动化交易工具平台

本页将设计文档转化为可浏览的产品架构说明。平台以 TradingAgents 作为研究决策引擎， 外层接入确定性的市场规则、账户风控、订单管理、仿真交易和审计回放能力。

设计原则

  • ✓LLM 只输出研究结论和解释，不直接下单。
  • ✓所有订单先经过市场规则和账户风控。
  • ✓A 股和港股分市场建模，统一交易抽象。
  • ✓先 paper trading，再半自动实盘，最后小额度自动化。

设计目标

把多 Agent 研究系统升级为面向 A 股和港股的自动化交易工具平台，但把交易安全边界交给确定性系统。

研究引擎

TradingAgents

保留分析师、牛熊研究员、交易员、风险经理和组合经理的多 Agent 协作链路。

交易内核

规则 + 风控 + OMS

将 Agent 输出转为强类型信号，再由规则和风控决定能否生成订单。

市场适配

A 股 + 港股

处理不同交易时段、手数、涨跌幅、T+1、港股 tick table 和 Stock Connect 约束。

TradingAgents 如何复用

原框架适合作为研究决策引擎，但不应直接拥有下单权限。

直接复用

LangGraph 编排、分析师角色、辩论机制、结构化输出、多 LLM provider、checkpoint、报告生成和记忆反思。

必须改造

数据源、A/H 专用工具、信号标准化、确定性风控、券商网关、订单生命周期和事件驱动回测。

总体架构

平台采用“研究引擎 + 市场适配 + 交易内核”的分层设计。

1数据层

行情、财务、公告、新闻、资金流、主数据。

2研究引擎

TradingAgents 多角色分析与辩论。

3信号标准化

Markdown 决策转为 SignalIntent。

4规则风控

市场规则、账户约束和策略阈值。

5OMS/EMS

订单状态机、切片、撤单和回报。

6审计回放

保存输入、输出、拒单原因和成交路径。

A 股与港股市场规则

规则引擎必须确定性实现，不能依赖 LLM 即兴判断。

市场 核心约束 平台处理方式
A 股 集合竞价、连续竞价、100 股手数、0.01 元 tick、涨跌幅、T+1 可卖。 按市场和板块建规则表，检查时段、手数、价格、涨跌停、停牌和可卖数量。
港股 开市前、连续交易、收市竞价、不同每手股数、动态 tick table、VCM/CAS。 按证券主数据加载 board lot 与 tick size，按订单类型检查可接受窗口和价格档位。
互联互通 标的名单、额度、交易日差异、北向/南向资金和订单窗口。 作为独立 eligibility 层进入市场规则引擎，失败时只给出解释，不生成订单。

模块设计

当前代码已经先落地 MVP 平台层，后续逐步补齐数据、回测、实盘网关和控制台。

数据中台

主数据、行情、公告、财务、新闻、舆情、宏观和资金流，生产环境应接授权数据源。

Agent 扩展

新增监管、流动性、公司行动、AH 溢价等分析角色，补充 A/H 股专用上下文。

信号服务

把 Buy / Overweight / Hold / Underweight / Sell 转换为可风控的结构化意图。

风控与执行

LLM 风控辩论提供解释，真正的交易拦截由确定性 RiskEngine 和 MarketRuleEngine 完成。

风控维度

最大单笔金额 最大仓位权重 最小置信度 可用现金 可卖数量 黑名单 行情异常熔断

执行方式

先使用 paper broker 验证流程。半自动实盘阶段默认进入人工审批队列，自动实盘只允许白名单策略、白名单账户和小额度。

核心对象

平台把研究结论转换为稳定的数据结构，降低自由文本进入交易系统的风险。

Instrument      # symbol, market, board, currency, lot_size, tick_size_rule
SignalIntent    # side, conviction, target_weight, entry_price, stop_loss
OrderIntent     # side, order_type, quantity, limit_price, tif
ValidationResult# accepted, issues, reject codes
Order           # status, filled_qty, avg_price, reject_reason

落地路线

从研究辅助到仿真，再到半自动实盘，最后进入小额度自动化。

阶段一

A/H 研究版

接入 A/H 证券解析、专用数据工具和中文报告，只输出报告与 SignalIntent。

阶段二

仿真交易版

实现 SignalNormalizer、MarketRuleEngine、RiskEngine、paper broker 和回测撮合。

阶段三

半自动实盘

接入券商网关，但所有订单默认进入人工审批队列。

阶段四

小额度自动实盘

仅对白名单策略、白名单标的和小额度账户开放自动执行。

安全边界

实盘交易风险来自数据、模型、执行、市场和合规多个方向，默认策略必须保守。

默认不开实盘。 当前平台优先服务 research-only 和 paper-trading。缺少数据授权、券商授权、合规审批、 人工确认和完整审计时，不应连接实盘账户自动下单。
